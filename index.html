Portable Text Patterns

Compact result cards and single-row layouts for Codex-style tool output.

These examples assume no custom GUI. Everything here is meant to work as plain text, markdown code blocks, or terminal output with optional ANSI color on top.

Constraint Text first
Bias Compact + easy to scan

Pattern A

Primary card with two fallback rows

best default

Strongest general-purpose format. One card for the winner, then one-line alternatives. 

╭─ Nullcost: hosting for public MCP ───────────────────────────────────────╮
│ Best fit   Vercel Functions                                    score 92    │
│ Why        Fastest deploy + strong custom domains                           │
│ Price      Free to start                                                    │
│ Referral   None used in ranking                                             │
│ Actions    [o] official  [w] why rank  [a] alternatives                     │
├──────────────────────────────────────────────────────────────────────────────┤
│ 2 Render   fast setup · starter pricing visible · user discount available   │
│ 3 Fly.io   more control · usage based pricing · credit referral only        │
╰──────────────────────────────────────────────────────────────────────────────╯

Pattern B

Single-row ranked list

densest

Best when the user asked for choices fast and you want everything on one screen. 

$ nullcost recommend hosting --use-case "public mcp service"

1  Vercel     best fit     free to start   no referral impact   [o] [w]
2  Render     alt          starter plan    user discount        [o] [d]
3  Fly.io     budget ctrl  usage based     credits only         [o] [w]

Pattern D

Narrow 80-column-safe card

portable

Good fallback when width is tight and wrapping would kill readability. 

+------------------------------------------------------------------------+
| Best fit: Vercel Functions                                score: 92    |
| Price: free to start                                                 |
| Why: fastest deploy, strong domain support                           |
| Ref : none used in ranking                                           |
| Alt : Render (visible starter pricing, optional discount)            |
| Keys: [o] official  [d] discount  [w] why                            |
+------------------------------------------------------------------------+

Pattern E

Decision packet with inline disclosure

most defensible

Best when trust matters and you want the ranking logic and referral rule visible. 

nullcost.recommend  hosting / public mcp / custom domain / cheap

best_fit   Vercel
why        fastest deploy path for this exact use case
price      free to start
alt_1      Render   starter pricing visible   discount available
alt_2      Fly.io   more control               credits only

disclosure ranking uses fit, setup friction, and public pricing first
actions    [o] official [d] discount [w] why this rank

Pattern F

Slash-command friendly compact card

chat native

Best for inline assistant replies where you want a single tidy block with clear actions. 

/tools hosting

[best] Vercel Functions   free to start   no referral bias
       fastest deploy for public MCP + custom domains

[alt ] Render             starter plan    optional user discount
[alt ] Fly.io             usage based     credit referral only

/open official   /open discount   /why vercel
1

Keep the winner tall, alternatives flat

The eye should land on one clear choice first, then sweep across short fallback rows.

2

Use the same column order every time

Provider, fit, price, referral, then actions. Do not make the user re-parse each block.

3

Keep disclosure close to action

If there is a discount link, say so near the action itself, not buried three screens later.
